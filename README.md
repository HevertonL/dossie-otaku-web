# Dossiê Otaku Web
Projeto de automação e engenharia de qualidade (QA) focado na criação de uma rede social de avaliações de animes.

## 🎯 Regras de Trabalho (Agile)

### 📌 Definition of Ready (DoR)
*O que uma tarefa precisa ter para entrar em desenvolvimento:*
- [ ] Título e descrição refletem o valor para o usuário final.
- [ ] Critérios de Aceite definidos.
- [ ] Contrato da API pré-definido (para tarefas de BackEnd).
- [ ] Issue atribuída ao responsável e com as Labels corretas.

### 🚀 Definition of Done (DoD)
*O que uma tarefa precisa ter para ser considerada finalizada:*
- [ ] O código compila e funciona localmente sem erros.
- [ ] A funcionalidade atende a 100% dos Critérios de Aceite.
- [ ] Rota/Regra de negócio coberta por testes de API (Supertest).
- [ ] Fluxo visual crítico coberto por teste E2E (Cypress).
- [ ] Nenhum bug de severidade "Alta" ou "Crítica" aberto.
- [ ] Pipeline do CI/CD (GitHub Actions) passando com sucesso.
